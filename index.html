This is the Git Training Website
  • git init --makes new repo
  • git add . or *.file --adds files to staging
  • git commit -m "comment" --commits staged files to repo the -m allows for comments(good practice)
  • git status -- gives status of edits, commits, adds, staging
  • git log -- shows one page of commits at a time
    • q can quit out of log
    • ctrl c (force quit)
    • git log --stat (gives +- for edits or changes as a visual)
    • git log --oneline --graph
  • git checkout switches branches where commits take place
    changes active branch
    • git checkout -b newbranchname makes new branch and checksout right away
    • git checkout commitname (can check out a commit)
      will make a detached head state not good to change code here, just use as
      visual for comparison
  • git clean -n -nx (x shows but does not delete)
    -f -fx
    -d -dx (directory)
      • Stashing Stash --used if not ready to commit, but also do not want to lose edits
    • snap shot of files
    • git stash push -m "working on example"

    • git revert (adds a new commit to older branch, but does not delete edits being done
    • git revert branchname
